How the feed works

01

Produce

POST events to /ingest from any source — sensors, APIs, webhooks, financial feeds. Batch up to 1000 events per request.

02

Chunk

Events are buffered in a time window (default 5s), serialised to JSONL, and uploaded as a verifiable blob to Shelby Protocol.

03

Index

Each chunk is indexed in Postgres with its sequence number, timestamp window, and schema fingerprint. Subscribers are notified instantly via Redis pub/sub.

04

Subscribe

AI pipelines subscribe via SSE and receive every new chunk the moment it lands. Or poll the REST API for historical replay.

Features
⚡ Sub-second delivery

Chunks land in subscriber pipelines within milliseconds of the window closing. No polling delays.

🔗 Decentralised storage

All chunk data is stored on Shelby Protocol — erasure-coded across 16 storage providers on private fiber.

📡 SSE subscriptions

Subscribe to any feed with a single HTTP connection. Receive ChunkRecord notifications in real time with auto-reconnect.

🔍 Historical replay

Query any past chunk by sequence number or timestamp. Stream blobs directly from Shelby RPC for training data replay.

🧬 Schema detection

Every chunk carries a schema fingerprint. Subscribers detect breaking changes automatically without a schema registry.
